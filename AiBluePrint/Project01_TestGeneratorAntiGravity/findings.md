# Findings

## Research
- Reviewed `Design.png` which shows a Chat layout (History Sidebar, Main Output, Input Field) and a Settings layout (Configuration fields for different AI providers, Save and Test Connection buttons).

## Discoveries & Requirements
- **Target of Generation:** Functional, non-functional, API, UI, and other types of test cases.
- **Inputs:** Users will either paste Jira card requirements or directly type requirement prompts in the chat.
- **Provider Support:** Needs configurable settings for Ollama API, LLM Studio API, Groq API, OpenAI API, Claude API, and Gemini API.
- **Tech Stack Backend:** Node.js using TypeScript.
- **Tech Stack Frontend:** ReactJS.
- **Formatting Constraint:** All test cases must be strictly formatted in a **Jira format**.

## Constraints
- **Protocol 0**: No scripts or code can be written until Discovery Questions are answered.
- **Protocol 0**: `task_plan.md` must have an approved Blueprint before execution begins.
